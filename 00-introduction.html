3  Introduction

AI Notebook is a powerful tool for processing both text and audio files using various AI models. It provides a structured way to interact with AI models, manage templates, and process files efficiently.

3.1 Overview

The AI Notebook system consists of:

  1. Core Processing Engine
    • Text and audio file processing
    • Multiple AI model support
    • Template-based instructions
  2. File Management
    • Structured project organization
    • Process tracking
    • Result visualization
  3. Model Integration
    • Support for multiple AI providers
    • Configurable model settings
    • Template-based prompting

3.2 Package Structure

ai-notebook/
├── src/
│   └── ainotebook/
│       ├── notebook.py      # Core processing
│       ├── cli.py          # Command line interface
│       ├── cli_quarto.py   # Visualization
│       ├── settings.json   # Model configurations
│       └── providers/      # AI model providers
│           ├── anthropic.py
│           ├── gemini.py
│           ├── openai.py
│           ├── groq.py
│           └── ollama.py
└── notebooks/             # Documentation
    ├── 00-introduction.qmd
    ├── 01-models.qmd
    └── 02-features.qmd

3.3 Project Structure

A typical AI Notebook project is organized as follows:

your-project/
├── notebooks/           # Source notebooks
│   └── analysis.qmd    # Contains AI blocks
├── process/            # Generated JSON files
│   └── analysis.json   # Contains responses
├── ai-views/           # Generated visualizations
│   └── analysis-vis.qmd
└── sources/           # Project resources
    └── templates/     # Jinja2 templates
        └── document.j2

For detailed information about specific features and capabilities, please refer to the following sections.
