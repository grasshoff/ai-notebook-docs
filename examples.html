---
model: claude-haiku
label: initial_analysis
---
Analyze the following text for key themes and arguments:

{{document}}

---
model: claude-sonnet
label: expert_analysis
---
Provide expert analysis of this topic.

---
model: gemini-pro
label: review
---
Review and enhance this analysis: {{expert_analysis}}

---
model: gpt-4
label: synthesis
---
Synthesize both perspectives:
1. Expert Analysis: {{expert_analysis}}
2. Review: {{review}}

7.4 Audio Processing

7.4.1 Basic Transcription

from ainotebook import process_audio

# Using default Gemini 2.0 model
transcript = process_audio(
    "lecture.mp3",
    model="gemini-audio",  # Uses Gemini 2.0 by default
    language="en"
)

# Using Whisper model
transcript = process_audio(
    "lecture.mp3",
    model="whisper",  # Explicitly use Whisper model
    language="en"
)

7.4.2 Advanced Audio Analysis

from ainotebook import process_audio, analyze_text

# Transcribe with Gemini 2.0 and speaker detection
transcript = process_audio(
    "interview.mp3",
    model="gemini-audio",
    language="en",
    speakers=True,
    speaker_count=2
)

# Analyze transcript with Claude
analysis = analyze_text(
    transcript,
    model="claude-haiku",
    template="interview_analysis.j2",
    variables={
        "focus": "key_insights",
        "style": "academic"
    }
)

# Multi-model analysis pipeline
results = {
    # Initial transcription with Gemini 2.0
    'transcript': process_audio("recording.mp3", model="gemini-audio"),
    
    # Backup transcription with Whisper for comparison
    'whisper_transcript': process_audio("recording.mp3", model="whisper"),
    
    # Analysis of both transcripts
    'analysis': analyze_text(
        "Compare these transcripts:\n\n1. {transcript}\n\n2. {whisper_transcript}",
        model="claude-sonnet"
    )
}

7.5 Project Organization

A typical AI Notebook project structure:

research-project/
├── notebooks/
│   ├── 01-data-collection.qmd
│   ├── 02-analysis.qmd
│   └── 03-conclusions.qmd
├── templates/
│   ├── analysis.j2
│   └── synthesis.j2
├── sources/
│   ├── data.csv
│   └── references.bib
├── process/
│   ├── data-collection.json
│   └── analysis.json
└── visualizations/
    └── analysis-dashboard.qmd

7.6 Command Line Usage

7.6.1 Processing Files

# Process a single notebook
ainotebook process notebook.qmd

# Process all notebooks in a directory
ainotebook process notebooks/

# Process without visualization
ainotebook process notebook.qmd --no-visualize

7.6.2 Creating New Projects

# Create a new project
ainotebook init my-research

# Create with custom name
ainotebook init path/to/project --name "Research Project"

7.6.3 Debugging

# Enable debug output
ainotebook process notebook.qmd --debug

# Create visualization with debug info
ainotebook visualize notebook.qmd --debug

7.7 Advanced Features

7.7.1 Custom Templates

{# analysis.j2 #}
Based on the provided {{style}} analysis approach:

1. Key Findings:
{% for finding in findings %}
   - {{ finding }}
{% endfor %}

2. Methodology:
{{ methodology }}

3. Conclusions:
{{ conclusions }}

7.7.2 Response Processing

---
model: claude-sonnet
process:
  - extract_key_points
  - format_citations
  - check_consistency
label: processed_analysis
---
Analyze the following research paper...

7.7.3 Visualization

from ainotebook import visualize

# Create interactive dashboard
dashboard = visualize.create_dashboard(
    data="process/*.json",
    metrics=["response_time", "token_usage"],
    interactive=True
)

# Generate comparison report
report = visualize.create_report(
    responses=["analysis_1", "analysis_2"],
    type="comparison"
)

These examples demonstrate the core features and flexibility of AI Notebook. For more detailed information about specific features, please refer to the Reference section.
