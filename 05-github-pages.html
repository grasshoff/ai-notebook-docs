6  Setting up GitHub Pages

This guide provides step-by-step instructions for setting up GitHub Pages for any ai-views directory, including creating individual SSH keys for repository access.

6.1 Prerequisites

  • Git installed on your system
  • GitHub account
  • Basic familiarity with terminal commands
  • Access to the ai-views directory you want to publish

6.2 Step 1: Create SSH Key

Each ai-views directory should have its own SSH key for secure GitHub access:

ssh-keygen -t ed25519 -C "friedrichstr@studio-3" -f ~/.ssh/id_rsa_ai_notebook

When prompted: 1. The key path is already specified in the command (-f option) 2. Enter a passphrase (recommended) or press Enter twice for no passphrase

6.3 Step 2: Add Key to GitHub

  1. Copy the public key:

    cat ~/.ssh/id_rsa_ai_notebook.pub

  2. Go to GitHub.com and log in

  3. Click your profile photo → Settings

  4. Select “SSH and GPG keys”

  5. Click “New SSH key”

  6. Give it a descriptive title (e.g., “AI Notebook”)

  7. Paste the public key content

  8. Click “Add SSH key”

6.4 Step 3: Test Connection

Verify the SSH connection:

ssh -T git@github.com

You should see a message like:

Hi username! You've successfully authenticated, but GitHub does not provide shell access.

6.5 Step 4: Configure Repository

  1. Navigate to your documentation directory:

    cd docu

  2. Initialize Git:

    git init
git branch -M main

  3. Add and commit your files:

    git add .
git commit -m "Initial documentation setup"

  4. Add remote (replace ‘username’ with your GitHub username):

    git remote add origin git@github.com:username/ai-notebook.git

6.6 Step 5: Enable GitHub Pages

  1. Push your repository to GitHub:

    git push -u origin main

  2. Go to repository settings on GitHub:

    • Navigate to Settings → Pages
    • Under “Source”, select:
      • Branch: main
      • Folder: /docs

6.7 Step 6: Deploy Documentation

  1. Ensure you’re in the documentation directory:

    cd docu

  2. Build the documentation site:

    quarto render

  3. Add the generated files:

    git add _site/*
git commit -m "Add generated documentation site"

  4. Push the changes:

    git push origin main

  5. Verify deployment:

    • Wait a few minutes for GitHub Pages to build
    • Visit https://username.github.io/ai-notebook
    • If needed, check the Actions tab on GitHub for build status

# Setting up GitHub Pages

This guide provides step-by-step instructions for setting up GitHub Pages for any ai-views directory, including creating individual SSH keys for repository access.

## Prerequisites

- GitHub account
- Basic familiarity with terminal commands

## Step 1: Create SSH Key

```bash
ssh-keygen -t ed25519 -C "friedrichstr@studio-3" -f ~/.ssh/id_rsa_ai_notebook
```

When prompted:
1. The key path is already specified in the command (-f option)
2. Enter a passphrase (recommended) or press Enter twice for no passphrase

1. Copy the public key:
