1 Introduction

AI Notebook is a tool for processing notebooks with AI-enhanced blocks and audio files. It supports multiple AI providers and offers a flexible template system for structured prompts.

2 Installation

pip install ainotebook

3 Setting Up a New Project

To create a new AI Notebook ensemble directory:

# Create project directory
mkdir my-ai-ensemble
cd my-ai-ensemble

# Create core directories
mkdir -p sources/templates process notebooks ai-views docs

# Initialize Quarto book configuration
cat > _quarto.yml << EOF
project:
  type: book
  output-dir: _site

book:
  title: "AI Analysis Ensemble"
  author: "Your Name"
  chapters:
    - index.qmd
    - notebooks/*.qmd
  sidebar:
    style: docked
    search: true
  page-navigation: true

format:
  html:
    theme: sandstone
    toc: true
    code-fold: true
    code-tools: true
    highlight-style: pygments
    page-layout: full
EOF

# Initialize settings
cat > settings.json << EOF
{
    "paths": {
        "process_dir": "process",
        "template_dir": "sources/templates",
        "source_dir": "sources",
        "notebooks_dir": "notebooks",
        "views_dir": "ai-views",
        "docs_dir": "docs"
    },
    "models": {
        "defaults": {
            "text": "claude-3-5-haiku",
            "audio": "whisper-large-v3"
        }
    },
    "providers": {
        "defaults": {
            "temperature": 0.1,
            "max_tokens": 8192,
            "language": "en"
        }
    }
}
EOF

# Create initial notebook
cat > notebooks/analysis.qmd << EOF
---
title: "AI Analysis"
format: html
---

\`\`\`{ai}
---
model: claude-haiku
---
Your first AI block
\`\`\`
EOF

# Create index file
cat > index.qmd << EOF
---
title: "Overview"
---

This is an AI analysis ensemble using AI Notebook.
EOF

This creates a sophisticated project structure:

  • sources/
    • templates/: Jinja2 templates for AI prompts
    • Other source files and data
  • process/: Processing results and intermediate files
  • notebooks/: Individual analysis notebooks
  • ai-views/: Generated visualizations
  • docs/: Documentation and notes
  • _quarto.yml: Quarto book configuration
  • settings.json: AI Notebook configuration
  • index.qmd: Project overview

The structure is designed as a Quarto book project, allowing you to:

  • Organize multiple analysis notebooks
  • Generate a cohesive documentation website
  • Keep source files and templates separate
  • Track processing results
  • Generate visualizations

4 Configuration

Required environment variables:

OPENAI_API_KEY=sk-...

ANTHROPIC_API_KEY=sk-ant-...
GROQ_API_KEY=gsk_...

5 Initialization

There are two ways to initialize AI Notebook:

5.1 1. Direct File Processing

from ainotebook import AINotebook

# Initialize and process a file
notebook = AINotebook("your_notebook.qmd")
notebook.process()

5.2 2. Manual Setup

from ainotebook import AINotebook

# Initialize without a file
notebook = AINotebook(debug=True)  # Enable debug mode for verbose output

# Later process a file
notebook.parse_file("your_notebook.qmd")
notebook.process()

5.3 Directory Structure

AI Notebook expects the following directory structure:

your_project/
├── sources/        # Templates and source files
├── process/        # Processing results
└── ai-views/       # Generated views

You can customize these paths in settings.json.
