5  GitHub Pages

This guide provides step-by-step instructions for setting up GitHub Pages for any ai-views directory, including creating individual SSH keys for repository access.

5.1 Prerequisites

  • Git installed on your system
  • GitHub account
  • Basic familiarity with terminal commands
  • Access to the ai-views directory you want to publish

5.2 Step 1: Create a New SSH Key

Each ai-views directory should have its own SSH key for secure GitHub access. This practice ensures better security and management of different repositories.

  1. Open your terminal

  2. Generate a new RSA key pair:

    ssh-keygen -t rsa -b 4096 -C "your-email@example.com" -f ~/.ssh/id_rsa_[project-name]

    Replace your-email@example.com with your GitHub email and [project-name] with a meaningful identifier.

  3. When prompted for a passphrase, you can either:

    • Press Enter twice for no passphrase (less secure but more convenient)
    • Enter a secure passphrase (recommended for sensitive repositories)

5.3 Step 2: Add SSH Key to GitHub

  1. Copy the public key content:

    cat ~/.ssh/id_rsa_[project-name].pub

  2. Go to GitHub.com and log in

  3. Click your profile photo → Settings

  4. Select “SSH and GPG keys” from the sidebar

  5. Click “New SSH key”

  6. Give it a descriptive title (e.g., “AI-Views [Project Name]”)

  7. Paste the entire key (including ssh-rsa at the start and the email at the end)

  8. Click “Add SSH key”

5.4 Step 3: Initialize Git Repository

  1. Navigate to your ai-views directory:

    cd path/to/your/ai-views

  2. Initialize Git repository:

    git init
git branch -M main

  3. Add your files:

    git add .
git commit -m "Initial commit"

5.5 Step 4: Create GitHub Repository

  1. Go to GitHub.com
  2. Click the “+” icon in the top right → “New repository”
  3. Name your repository appropriately (e.g., “ai-views-[project-name]”)
  4. Add a description (optional but recommended)
  5. Choose “Public” visibility for GitHub Pages functionality
  6. Do not initialize with README, .gitignore, or license
  7. Click “Create repository”

5.6 Step 5: Connect and Push to GitHub

  1. Add the remote repository:

    git remote add origin git@github.com:username/repository-name.git

    Replace username and repository-name with your GitHub username and repository name.

  2. Push your code:

    git push -u origin main

5.7 Step 6: Enable GitHub Pages

  1. Go to your repository on GitHub
  2. Click “Settings”
  3. Scroll to “GitHub Pages” section
  4. Under “Source”, select:
    • Branch: main
    • Folder: /docs
  5. Click “Save”

5.8 Step 7: Configure Quarto for GitHub Pages

  1. Update your _quarto.yml file:

    project:
  type: website
  output-dir: docs

website:
  title: "Your Project Title"
  navbar:
    left:
      - href: index.qmd
        text: Home

format:
  html:
    theme: cosmo
    css: styles.css
    toc: true

  2. Build your site:

    quarto render

  3. Push the generated docs:

    git add docs/
git commit -m "Add generated site"
git push

5.9 Maintenance and Updates

To update your GitHub Pages site:

  1. Make changes to your content

  2. Render the site:

    quarto render

  3. Commit and push changes:

    git add .
git commit -m "Update site content"
git push

5.10 Troubleshooting

5.10.1 Common Issues

  1. SSH Key Permission Denied
    • Verify key is added to GitHub
    • Check key permissions: chmod 600 ~/.ssh/id_rsa_[project-name]
    • Test connection: ssh -T git@github.com
  2. Push Rejected
    • Pull latest changes: git pull origin main --allow-unrelated-histories
    • Resolve conflicts if any
    • Push again: git push origin main
  3. GitHub Pages Not Updating
    • Check if docs folder is in the correct location
    • Verify GitHub Pages settings in repository
    • Wait a few minutes for changes to propagate

5.10.2 Getting Help

If you encounter issues:

  1. Check Git error messages
  2. Review GitHub Pages documentation
  3. Consult the AI Ensembles team
  4. Open an issue in the repository

5.11 Best Practices

  1. Use meaningful commit messages
  2. Keep repository clean and organized
  3. Regularly update content
  4. Test locally before pushing
  5. Maintain clear documentation
